Verbindung einrichten

Mit einem per E-Mail erhaltenen API-Key ist Agentic-Firmenbuch.at in wenigen Minuten im KI-Tool verbunden. In allen Beispielen DEIN_API_KEY durch den eigenen Schlüssel ersetzen.

0 Voraussetzung

  • Ein API-Key aus der Bestätigungs-E-Mail. Noch keiner? Hier kostenlos anfordern.
  • Server-Adresse: https://mcp.agentic-firmenbuch.at/mcp

1 Claude Code Terminal

Nur für Claude Code (das Terminal-Tool). Für die Desktop-App gilt stattdessen Schritt 2. Ein Befehl, als eine Zeile einfügen (macOS, Linux, Windows/PowerShell identisch):

Terminal öffnen: macOS → ⌘ + Leertaste, „Terminal" tippen · Windows → Startmenü, „PowerShell" · Linux → das jeweilige Terminal-Programm.

claude mcp add --scope user --transport http agentic-firmenbuch https://mcp.agentic-firmenbuch.at/mcp --header "X-API-Key: DEIN_API_KEY"

Verbindung testen: einen neuen Chat öffnen und z. B. fragen: „Bist du mit dem Agentic-Firmenbuch-Server verbunden, und welche Werkzeuge stellt er bereit?" – nennt der Agent die Firmenbuch-Werkzeuge (Unternehmen suchen, Kennzahlen, Historie …), ist die Verbindung aktiv.

2 Weitere MCP-Clients VS Code · Cursor · …

Jeder MCP-Client, der entfernte HTTP-Server mit Header-Authentifizierung unterstützt – VS Code (GitHub Copilot), Cursor und weitere – verbindet sich mit denselben zwei Angaben:

  • Adresse: https://mcp.agentic-firmenbuch.at/mcp · Typ http
  • Header: X-API-Key: DEIN_API_KEY

Die genauen Schritte stehen in der offiziellen Doku des Tools: VS Code (MCP) · Cursor (MCP). Üblicherweise als mcp.json-Eintrag:

{
  "mcpServers": {
    "agentic-firmenbuch": {
      "type": "http",
      "url": "https://mcp.agentic-firmenbuch.at/mcp",
      "headers": { "X-API-Key": "DEIN_API_KEY" }
    }
  }
}

Hinweis: Diese Clients sind MCP-konform; die genaue Bedienung folgt der jeweiligen Hersteller-Doku.

5 Verfügbare Werkzeuge (MCP-Tools)

Der Server stellt dem KI-Tool diese Werkzeuge bereit. Der Agent wählt sie selbst passend zur Frage – ein direkter Aufruf ist nicht nötig.

  • search_companies – Unternehmen filtern (Region, Rechtsform, Größenklasse, Bilanzsumme, Eigenkapitalquote, Umsatz, Mitarbeiter, Wachstum, GF-Alter) → Liste kompakter Treffer-Karten.
  • get_company_details – kuratiertes Firmenprofil zu einer Firmenbuchnummer.
  • get_company_history – Mehrjahres-Zeitreihen je Kennzahl (Bilanzsumme, Eigenkapitalquote …).
  • get_full_record – vollständiger Datensatz inkl. aller Bilanzpositionen samt offizieller UGB-Codes.
  • get_document – Referenz/Link zum Original-Jahresabschluss.
  • list_sectors – Branchen-/Größen-Übersichten mit Anzahl.
  • get_cohort_summary – Aggregat-Kennzahlen einer Kohorte (z. B. alle GmbHs in der Steiermark).
  • find_peers – vergleichbare Unternehmen nach Größenklasse und Bilanzsumme.

6 Welche Daten geliefert werden

Pro Unternehmen, strukturiert und mit Herkunftsnachweis:

  • Identität – Firmenname, Firmenbuchnummer, Rechtsform, Status (aktiv/historisch/gelöscht).
  • Standort – Bundesland, Sitz, Firmenbuchgericht.
  • Größe – Größenklasse (Kleinst/Klein/Mittel/Groß), Bilanzsummen-Band.
  • Finanzen – Bilanz- und (wo veröffentlicht) GuV-Werte, mehrjährig; jede Position mit ihrem offiziellen UGB-Code (z. B. „§ 224 Abs 2 A II").
  • Kennzahlen – Eigenkapitalquote, Verschuldung, Deckungsgrade, EBIT/EBITDA, Margen, ROE/ROA – berechnet, mit Verlauf.
  • Wachstum – Profil (schrumpfend/stabil/wachsend/stark wachsend) + Raten.
  • Mitarbeiter – wo veröffentlicht.
  • Geschäftsführung – Name, Funktion, Vertretungsart sowie Alter bzw. Geburtsjahr (nur Jahr, kein Tag/Monat – DSGVO).
  • Originaldokumente & Herkunft – Liste der Jahresabschlüsse + Quelle und Lizenz (CC BY 4.0).
