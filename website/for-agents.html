MCP server

Built for AI agents

Agentic-Firmenbuch serves the official company and financial data of the Austrian company register (Firmenbuch) as an MCP server. Your agent asks in natural language, the server returns structured, filtered, and aggregated data on over 340,000 companies.

Endpoint and connection

One-click (OAuth)

Claude web, desktop, and Cowork. Add a connector, confirm your email, done. No API key needed.

Address: https://mcp.agentic-firmenbuch.at/mcp

API key (header)

Claude Code, Cursor, VS Code. Use the endpoint as an http transport with your key from the email:

Endpoint: https://mcp.agentic-firmenbuch.at/mcp
Header:   X-API-Key: YOUR_API_KEY

Cowork and an API key use the same account. Full per-client guide at onboarding.en.html.

Quickstart in 3 steps

  1. Get access. Verify your email (Free) or subscribe to Pro. Your API key arrives by email; for Claude Cowork you need none.
  2. Connect your client. Point your MCP client at https://mcp.agentic-firmenbuch.at/mcp - via OAuth click or with the header X-API-Key: YOUR_API_KEY.
  3. Ask in natural language. For example: "Show me mechanical-engineering companies in Styria with over 50 employees and compare equity ratio and revenue growth."

Available tools

ToolDescription
search_companiesSearch and filter companies by name, industry, Bundesland, size or financials, sorted and paginated.
get_company_detailsFull financial profile of one company: multi-year balance sheet and P&L, ratios, growth, management.
get_full_recordThe complete record for one company, the full superset we hold.
get_company_historyMulti-year time series of individual metrics for one company.
get_cohort_summaryAggregate statistics over a whole cohort or industry of companies.
find_peersFind the companies closest in size to a given one.
get_coverageHow much of the register we actually serve, as an aggregate dashboard.
get_documentA time-limited download link to a company's official annual filing.
list_sectorsValid filter values for search (industries, legal forms) with company counts.
describe_fieldsCatalog of every field the server returns, with code tables and null rules.
get_my_usageYour own API-key usage: call count and weighted compute-units per tool.

Full field and code reference: felder.en.html. The server does not understand language itself, it filters and aggregates deterministically, your agent translates the question into these tools.
